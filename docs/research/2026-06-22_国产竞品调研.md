# 国产 Agent / LLM 可观测性竞品调研

> 日期：2026-06-22
> 缘起：[`docs/analysis/2026-06-22_竞品分析.md`] 指出最大盲区——海外 SaaS 结构上进不了气隙金融机房，**真正能威胁 yiTrace 的是国产玩家**，但此前调研全集中在海外。本文专项补这块。
> 方法：联网检索厂商公开文档/官宣/技术博客（2025–2026），按公开信息归纳，非逐家逐版本核对；私有化/合规等签单细节以厂商口径为准，需销售侧再核。
> 一句话结论：**国产玩家分三类——云厂商（强但云端绑定）、APM 老兵（私有化在手、正吞 LLM，最贴身）、开源 Agent 平台（可观测外包、自己不做存储）。没有一家同时做到「中文原生检索 + 自有语义召回 + 单机自有引擎」，但 APM 老兵的私有化金融客户根基是我方最该警惕的护城河缺口。**

---

## 1. 为什么单独调研国产

海外标杆（Langfuse / LangSmith）在金融政企气隙场景里被合规红线挡在门外——这是结构性出局，不是功能差距。于是真正在客户那张采购清单上和我们并列的，是国产玩家。它们的优势恰恰是我们要立的那几条：天然数据合规、本地化生态、私有化交付经验、已有的金融客户关系。把它们摸清，才知道我方差异化到底剩多少。

## 3. 类一 · 云厂商自带可观测

### 3.1 阿里云百炼（Model Studio）+ ARMS

| 项 | 情况 |
|---|---|
| 形态 | 云端。百炼应用观测能力**架在 ARMS 可观测平台**上，聚焦 LLM 领域洞察 |
| 能力 | 端到端看 LLM 应用全链路（向量生成、向量检索、大模型调用），分钟级延迟/token 指标；用 session 串问答、trace 承载全链路节点；定义 LLM Span Kind（CHAIN / EMBEDDING / LLM 等）领域语义 |
| 埋点 | 自研 Python Agent，支持最新 OTel LLM 语义约定 + LlamaIndex/LangChain/通义/OpenAI，框架 Callback + 装饰器无侵入埋点，自动采 Metrics/Traces/Profiling |
| 对我方威胁 | 技术成熟、OTel 原生、生态完整，但**深度绑定阿里云 ARMS/百炼**，私有化/气隙不是它的主场；中文检索/自有语义召回非其卖点（它把存储交给 ARMS 后端） |

### 3.2 火山引擎 APMPlus + 全栈可观测平台

| 项 | 情况 |
|---|---|
| 形态 | 云端。**OneAgent 统一探针**，从 App/Web/小程序 → AI 网关 → Agent 工具 → 底层 LLM 全链路 MTL 统一采集 |
| 能力 | LLM Trace 分析、LLM 会话分析、vLLM 推理引擎链路监控；提出「从 Trace 到回放，构建生产级智能体全链路观测」；eval 用 **LLM-as-Judge**（相关性/幻觉/任务完成度/安全）；每会话成本 + token 利用率 |
| 生态 | 与字节系框架强绑（CloudWeGo、开源 Eino 框架、Coze、豆包） |
| 对我方威胁 | Agent 可观测叙事最完整的国产玩家之一（已喊「回放」），但**云端、字节生态绑定**；公开资料未见中文倒排/带过滤向量召回这类存储级差异化 |

### 3.3 腾讯云 ADP（智能体开发平台）/ TI 平台

| 项 | 情况 |
|---|---|
| 形态 | 云端。ADP 提供 LLM+RAG / Workflow / Multi-agent 多种 agent 框架，主打**用自有数据建稳定安全的 agent** |
| 能力 | 领先的 Agentic RAG 跨文档检索、全链路监控告警 + 数据洞察；LLM 可观测需端侧 trace 采集 + 服务端上报，支持 LangChain / 最新 OTel 语义 |
| 对我方威胁 | 知识引擎 / Agentic RAG 是强项，但可观测是**平台附属能力**，非独立产品；云端为主 |

**类一小结**：技术都成熟、OTel 原生、生态强，但**本质是云厂商拉新和锁定的附属能力，私有化气隙金融不是主战场**；且都把存储交给自家后端（ARMS / ByteHouse 等），结构上同样挂不进中文倒排 + 图式向量这种自定义一等索引。它们的威胁是「客户已经在用这朵云，顺手就用了」，不是「单机私有化做得比我们好」。

---

## 4. 类二 · APM 老兵下场吞 LLM（★最贴身威胁）

这是离 yiTrace **最近的威胁**：它们手里已经握着我们想立的两张牌——**私有化交付经验 + 金融政企客户根基**，现在正把 LLM 可观测往里塞。

### 4.1 博睿数据 Bonree ONE

| 项 | 情况 |
|---|---|
| 底子 | 国产 APM 头部，Bonree ONE 一体化智能可观测平台，全栈全端（DEM/APM/ITIM/NPM），无监督根因分析 + 自适应生成式 AI；**有大量私有化金融政企客户** |
| LLM 动作 | 2025 春季版「云原生适配 + LLM 大模型接入」为核心亮点；**全面接入 DeepSeek**，提「可观测性 + DeepSeek 深度融合，助力企业实现私有 LLM 服务全生命周期可观测」；公开过「LLM 在可观测体系建设落地的 30 大核心技术应用场景」 |
| 对我方威胁 | **最高**。它本就在金融机房里、本就做私有化、本就懂等保合规，现在只差把 Agent trace 那套语义补全。它的短板是 LLM 可观测起步晚、是在通用 APM 上加 LLM，而非为 Agent trace 原生设计（无中文倒排/带过滤语义召回这类专门能力） |

### 4.2 基调听云

| 项 | 情况 |
|---|---|
| 底子 | 国内最早提 APM + 用户体验优化的厂商，**五次入选 Gartner APM 魔力象限**；一体化监控 + 用户体验 + DevOps + 基于 AI 的智能可观测 |
| LLM 动作 | 向 AI 智能可观测演进，公开资料里 Agent/LLM trace 的专门能力披露少于博睿 |
| 对我方威胁 | 中高。品牌与金融客户在，但 LLM/Agent 方向叙事不如博睿激进 |

### 4.3 云智慧

通用可观测 + AIOps 路线，私有化交付能力强，LLM 可观测亦在跟进，威胁等级与听云相近。

**类二小结**：**它们的护城河正是我方想立的「私有化 + 金融客户 + 合规」**——这恰恰打在我方已明确暂缓的等保/RBAC/加密软肋上。但它们是「通用 APM 加 LLM 皮」，缺的是我方押注的存储级原生能力：中文倒排 BM25、带过滤图式语义召回、审计级确定性 event_id 账本。**正面交锋点 = 谁能把「Agent trace 专用、中文原生、审计账本」做到它们结构上补不动的深度，同时尽快补齐它们已有的合规门槛。**

## 5. 类三 · 开源 Agent 平台（可观测外包）

### Dify / Coze（开源版）/ FastGPT

- **Dify**：开源 LLMOps 平台，含可视化工作流、RAG、Agent、多模型接入、可观测运维。关键点——**它的可观测是「集成外部工具」：v0.6.12 起对接 LangSmith 和 Langfuse**，自己不造 trace 存储。私有化部署能力强。
- **Coze（扣子）开源版**：微服务架构（Go + React/TS），**支持私有化部署**满足企业数据安全；可观测同样偏向接外部或平台自带。
- **FastGPT**：开源 RAG/知识库为主，可观测偏轻。

**类三小结**：这群是**潜在渠道而非正面对手**。它们把可观测外包出去——既然能接 LangSmith/Langfuse，理论上也能接一个**国产、私有化、中文原生**的 trace 后端。**Dify/Coze 私有化部署 + yiTrace 当它们的可观测后端**，是一条可探索的集成路线（客户已经私有化部署 Dify，可观测却只能接出境的 Langfuse，这正是我方切入口）。

## 7. 关键判断

1. **真正的对手在国产、且分工明确**：云厂商靠生态绑定（云端，私有化非主场）、APM 老兵靠私有化金融根基（最贴身）、开源平台靠交付但外包可观测（潜在渠道）。海外标杆在金融气隙场景几乎不构成直接竞争。
2. **最贴身威胁 = 博睿/听云这类 APM 老兵**：它们已在金融机房、已做私有化、已接大模型。我方对它们的软肋（合规/客户根基）正是它们的强项，我方的强项（中文原生 + 带过滤语义召回 + 审计账本 + Agent trace 原生）正是它们「通用 APM 加 LLM 皮」补不动的深度。**这是主战场。**
3. **没有一家占了我方那个格子**：中文原生检索 + 自有语义召回 + 单机自有引擎，三类国产玩家全空——云厂商把存储交给自家后端、APM 老兵在通用栈上加 LLM、开源平台外包可观测。**我方差异化在国产语境下依然成立**，但窗口取决于博睿这类玩家补 Agent trace 专用能力的速度。
4. **开源 Agent 平台是渠道不是敌人**：Dify/Coze 私有化部署却只能接出境的 Langfuse 做可观测——**「私有化 Dify/Coze + 国产中文原生 trace 后端」是一条现成切入路线**，值得做 PoC 级集成验证。

---

## 8. 对路线的建议

1. **把竞争叙事从「打海外」切到「打国产 APM 老兵」**：销售话术要能回答「比博睿强在哪」——答案是 Agent trace 原生 + 中文倒排 + 带过滤语义召回 + 审计账本，这些是它们结构上加不动的。
2. **合规要排进路线，不能一直暂缓**：等保/RBAC/加密是国产 APM 老兵的标配、是金融签单门槛。我方在这条线上落后于全部类二玩家，越晚补越被它们用「合规齐全」挤出局。
3. **做一条 Dify/Coze 集成 PoC**：验证「私有化开源 Agent 平台 → 接 yiTrace 当中文原生可观测后端」，把类三从竞品变渠道。
4. **持续盯博睿/火山的 Agent trace 进展**：这两家叙事最激进（博睿私有化金融、火山已喊回放），是后续季度复盘的重点监控对象。
5. **情报仍有限**：本文基于公开资料，厂商私有化金融案例的真实深度、合规清单、价格需销售侧实地核。建议建立国产竞品季度跟踪表。
