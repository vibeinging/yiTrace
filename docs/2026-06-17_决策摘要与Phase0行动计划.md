# yiTrace 决策摘要 + Phase 0 行动计划 **(v2 · 经对抗性压测修订)**

> 日期：2026-06-17｜面向：决策层（CEO / 技术负责人 / 商务 / 董事会）。
> v1 经三视角对抗性压测（执行/投资人/诚实审计，均判 `risky`），本 v2 把暴露的乐观全部改掉。**给董事会的文档，乐观即失职。**

## 1. 钱与 runway（v1 缺，董事会必看）

> 数字是**量级估算**，须由你们用真实薪资/客户数填实；这里给的是"董事会会问、计划必须答"的框架。

- **零回款期烧钱**：13-16 人 × 国内复合人才年成本粗估 ~2000-3500万/年 × (28-40月到首单) ≈ **5000万 - 1.1亿**，全程零 license 回款。
- **单客户 ACV 锚点**（竞品，**待一手核实**）：LangSmith 企业自托管 ~$100K+/年（无公开报价、推断）；Langfuse 自托管原 $3-4K/月 —— 但**已被 ClickHouse 收购、现免费 MIT 自托管**（这把"私有化省钱"卖点正被免费强敌抹平）。
- **必须由董事会拍的三个数**：① 母体（yiTrace/AgenticData 是否盈利）能撑多长 runway；② 几单回本、目标客户池有多大（金融政企愿私有化采购 Agent 可观测的真实数量）；③ **止损阈值**——什么信号出现就转向/砍线（如 Phase 0 CESI 口径要求重自研、或语义召回付费意愿验不出、或 18 月仍无首个付费 PoC）。
- **互联网/中小双轨 = 认证等待期唯一现金流 + PMF**（见 §3），它不是"兜底"，是 runway 模型的一部分，越早起跑越摊薄烧钱。

## 2. 真实时间线（串行链，非并行）

```
[招聘 3-6月] ─► [dogfood+引擎+平台 MVP 成熟 ≥13月] ─► [应用层信创/等保/密评 测评 3-6月] ─► [排 CESI/安可批次 一年2次, 最坏+6月]
   │                    │（平台 60-70% 工时在此，v1 未细化）        │（测评客体=成熟产品，Phase0 无产品可测）
   └ Phase 0 能起跑的只有【不依赖产品】的认证子段：软著/IP固化、CPU×OS×openGauss 底座栈适配互认证、抢 CESI 批次报名占位
═══════════════════════════════════════════════════════════════════════►  金融政企首单 ≈ 28-40 月
∥ 互联网/中小双轨（不要信创，认证等待期跑收入+PMF+打磨语义召回钩子）─────────────────────────►
```
**关键修正**：信创认证不是一根从第0周并行到第18月的独立长杆；它的**真正测评段串在产品成熟之后**。"认证 Phase 0 起跑"只能启动底座适配/软著/抢批次这些**不需要产品**的部分。
**团队**：13-16 人，但**其中 ~10-12 人（平台前后端/摄入/向量/私有化）现在还不存在，要招**——招聘期 3-6 月 + 磨合 1-2 月**必须进时间线**，且 Phase 0 工程 PoC 需要的内核/向量人若还在招，Phase 0 本身就起步即延期。

## 3. Phase 0（重投入前的真门禁；商务=慢变量，工程+市场=快变量）

> 修正：v1 把 Phase 0 压成"1-4 周"且只测语法/机制。v2 拆成三类并行，**承认商务是 2-6 月慢变量**，并把"市场/付费验证"和"并发正确性回归"提为一等门禁。

### A. 商务/合规（**立即启动、2-6 月闭环**；gate 住"去不去外国开源/上不上重自研"，绝不因它慢提前拍板）
1. 拿 ≥1 份目标金融政企**真实采购需求书**（定 HA/信创/等保/密评是否准入前置 + 量级 + 期限）。
2. 向 CESI/测评机构**书面确认**：应用软件信创认证是否对"关键路径用外国开源"判负、有无自研占比硬阈值。**→ 大概率"开源可接受"，则 DuckDB/Lance 可用、省 ~18 月自研。这是承重墙，目前是空的，必须坐实。**
3. 信创/等保三级/密评真实周期+报价（含 yiTrace 须独立走的那份应用层认证）。
4. **一手核实"窄口空白"**：阿里云/火山/字节是否已对金融政企出私有化 Agent 可观测（附录 C：三家均有生产级、阿里牵头信通院国标）。

### B. 工程 PoC（一台 openGauss-yiTrace 实例，**1-4 周，纯本地不依赖外部**，但需已招到内核/向量人）
- **招牌钩子 recall 实测**（最高优先）：DiskANN 带**任意元数据列**（tenant/span_kind/time）过滤召回的精确列序/opclass/WITH 参数 + **recall 质量 + 高选择度 under-fill**。源码只证明 id-范围过滤在 HA 测试跑过，**任意元数据列召回从未验证**——这是整个差异化定位的前提。
- **中文 BM25 建索引可用**：`USING fulltext` 复合列 + `DICTS#ALGORITHMS#COEFFICIENTS` 在你们这版二进制能否跑通（全 src/test 树 0 回归测试，只 doc 记载且带 TODO）。
- **折叠吞吐 benchmark**（1亿/天单机，query_dop=1）→ **决定"v1 不动内核"是否成立**；扛不住=触发 kernel 算子=动内核=团队/时间线变重。
- **并发正确性回归**（不只 DDL 原子性）：附录 N 已判调度链 risky，列出 4 条静默丢数据路径（ingest_ts 水位、event_id 非单调、freeze TOCTOU、remelt 从有损 CStore 重建）——**修复版已写进调度文档，但必须跑回归证明"晚到+乱序"不丢事件**。金融政企对静默丢 trace 零容忍（审计可复现=准入）。
- DDL-in-TX 原子性、SKIP LOCKED+ASTORE、event_id 真单调雪花、MERGE/ON DUPLICATE+WHEN MATCHED WHERE、查询层动态索引名、国产 CPU 上 Qwen3-0.6B-int8 吞吐。

### C. 市场/付费验证（**零成本、不依赖产品成型、提前到 Phase 0**）
- **语义召回付费意愿**（招牌钩子值不值得建）：用现有 AgenticData/yiTrace 客户 + 最糙 demo/landing page 做四级验证前两级（使用率埋点、加价档转化意向）。**若市场说"trace 能查就行、语义召回可有可无"→ 整个"窄而深"定位塌方**——这必须在花 4-6 月建引擎之前问。
- **互联网/中小双轨起跑**：认证等待期唯一现金流+PMF+产品打磨来源，越早越摊薄烧钱。

## 4. 必须正面回答的战略问题（v1 回避了，董事会会问）

1. **为什么是我们，不是阿里/华为/Doris 顺手做？** 三家已有生产级 Agent 可观测；**openGauss 是华为 IP**——用华为内核做信创护城河，等于把叙事控制权交给一个能顺手做掉你的竞品。"守窄而深"成立的前提是巨头不顺手覆盖这个窄口——**必须论证或证伪**。
2. **引擎还是平台？** 金融政企买的是**平台**（trace 浏览器/eval/告警/RBAC/看板/SDK 矩阵/离线升级诊断/气隙 embedding），不是存储引擎。这 60-70% 工时**必须进路线图/团队/预算**，不能 gated 到 Phase 0 之后推迟定义产品大头。一个穿薄服务外壳的存储引擎，在比预算低一层的位置竞争。
3. **护城河刷新到 2026-06 现状还成立吗？** LangSmith 已有 similar_examples/dynamic few-shot（语义）；ClickHouse-Langfuse V4 免费 MIT + 训练集导出 + eval 飞轮。五飞轮原语里四个是 table stakes，**唯一稀缺的"带过滤语义召回"是 ClickHouse 系一个 sprint 能补的特性层**。差异化必须收敛到可证伪的工程优势（单机本地 NVMe 更低延迟 + 原生中文 + 数据不出境 + 信创主体资格），别押"对手补不上的结构性空档"。
4. **单元经济撑得起一条线吗？**（见 §1）

## 6. 我的诚实建议（conditional go，不是 go）

**不建议现在全力投入。建议先做一个 ~2-6 月、几千万级以下的 Phase 0 决策门**，门后才是 go/no-go：

1. **本周**：启动 Phase 0 A（商务核实，尤其采购需求书 + CESI 书面口径）+ C（语义召回付费验证 + 互联网双轨起跑）+ 信创底座适配抢批次。**这些零成本/低成本、却能翻盘最重决策。**
2. **1-4 周（招到人后）**：Phase 0 B 工程 PoC，**先验招牌钩子 recall + 折叠吞吐 + 并发正确性**——它们决定"窄而深定位"和"不动内核"是否成立。
3. **Phase 0 门评审**（~2-6 月）：拿 A/B/C 结论，回答 §4 四个战略问题，重算时间线/钱，再做 **go / no-go / 转向（如转 OEM 引擎、或先做不要信创的互联网版、或砍线）**。
4. **若过门**：平台层细化（产品大头）+ 摄入双轨 + dogfood，金融政企版与认证测评段对齐。

> **一句话**：技术地基我们已经挖到 DDL 和并发 bug 级、且去了风险（不动内核、schema/调度已修十几个洞）；但项目的成败不在技术深度，而在 **§3 那几件零成本却能翻盘的核实有没有先做、§4 那几个战略问题答不答得了、§1 那笔钱董事会愿不愿烧 28-40 月**。**先过 Phase 0 决策门，再谈全力投入。**
