# 向量索引：磁盘型图索引（可落盘），参考 yiTrace graph_index

> 2026-06-23 ｜ 现状索引见 [CURRENT_STATE.md](../CURRENT_STATE.md)。
> 用户要点：**HNSW 是内存型的；我们要能落盘**。参考 yiTrace-Lite 的 `graph_index`（在
> `yitrace-vector/src/gausskernel/storage/access/hnsw/`，pgvector 衍生 + 向量单独缓冲的改良版）。

## 1. 参考：graph_index 的落盘设计（从内核代码 + 用户文档读出）

graph_index = 原生 HNSW 改良，**解决 HNSW 内存占用过高**（1000w×1024 维原生要 150~300GB 内存）。三招：

1. **元页**（`HnswMetaPageData`，block 0）：magic/version、dimensions、m、efConstruction、
   **entry(blkno/offno/level)**、insertPage、num_vectors、metric。
2. **节点元组分页存**（`HnswTupleData`）：`level` + **`deleted` 软删标记** + heaptids（外部行指针）
   + **`floatVectorIndex`（向量槽号，指向独立向量存储）** + `neighbors[]`（每个邻居 = 目标节点的
   indexTid(blk/off) + 其向量槽号）。元组**不内联向量** → 页里能塞更多节点 → 遍历触达更少页。
3. **向量单独存 + 缓冲**：向量放 `vector_buffers` 专用缓冲池，邻边/元数据走 `shared_buffers`。
   **遍历图只需邻边+元数据（小、热），向量按需取（大、冷）**——这是比 pgvector（向量内联元组）省内存的关键。

附带：软删（`deleted`）+ 文档建议"删/改超 40~60% 重建一次"；按机器架构动态选距离指令集（SIMD）。

## 2. Rust 版设计：`DiskGraphIndex`（落进引擎，std-only）

实现引擎 `GraphIndex` trait（`index_embedding` / `search(query,k,filter)`），与现有 `segstore.rs` /
`vecstore.rs` / `persist.rs` 同类。**不照搬 openGauss 页式（BLCKSZ/ItemPointer/MVCC/buffer manager 那套
为内核设计的重东西）**，吸收其核心三招：

```
一个索引目录：
  meta            —— 入口点(node_id/level)、m、ef、dim、metric、节点数、删除数
  nodes           —— 节点元组分页：node_id → { level, deleted, ext_id(trace,span), vec_slot,
                      neighbors[level][] }（只存图结构，小）
  vectors         —— 向量分页：vec_slot → f32[dim]（复用/扩展 vecstore，大，按需读）
  + 内存缓冲池（LRU page cache）盖在 nodes/vectors 上，搜索按需读页，不全量常驻
```

- **落盘**：插入写节点页 + 向量页 + fsync；元页原子更新入口点。**重启从盘恢复、不全量 rebuild**
  （现有内存 NSW 的 `graph.rs` 是重启 rebuild，这正是要改掉的）。
- **进图过滤（护城河）**：搜索时 `filter` 驱动收点/停止、导航穿过不满足的点（同 `graph.rs` 实证）。
- **软删**：删除打 `deleted` 标记，搜索跳过；累计超阈值触发重建（先留 stat，不自动重建）。

## 3. 工作负载特征（用户确认，定取舍）

- **删除少** → 软删（标记 + 搜索跳过）足够；不投入 tombstone 压缩/vacuum，"死元组多了重建"几乎用不上。
- **append 极多** → **插入吞吐第一**：插入**只写不刷**（写页缓存，便宜），靠提交点 `flush()` 批量 fsync
  （像 WAL 组提交）。`GraphIndex::flush()` 接进引擎 `persist_manifest`，append 期间零 fsync。

## 4. 分阶段落地（每阶段可测可提交）

1. ✅ **持久化基座**：分页节点存储 + 向量存储 + 元页 + 字节预算缓冲池（`vector_cache_bytes`，对齐
   `vector_buffers`，例 1GiB）；**重启不 rebuild**。
2. ✅ **图导航**：单层 NSW 落盘版建图（连边 + 反向边 + 度数剪枝）+ beam search 按需读页（邻边读节点、
   距离按需读向量走缓冲）；**进图过滤天然成立**（beam admit 驱动停止 + 穿过不满足点）。召回@10 ≥ 0.85。
3. ✅ **接进引擎**：`DurableGraphIndex`（首向量定维度的惰性磁盘索引）成为 `open_durable` 默认 `GraphIndex`，
   不用 vecstore、recover 不 rebuild；`flush()` 挂提交点。端到端测过。参数 `DiskGraphConfig`
   （m / vector_cache_bytes / ef_construction / ef_search）。
4. ✅ **多层 HNSW**（顶层导航）：节点 level 指数分布（由 node_id 确定性算）、**底层(0)+向量在磁盘、上层图稀疏
   常驻内存 + 快照持久**（`upper` 文件，crc+原子写）、入口=最高层节点。查询顶层贪心下沉→底层 ef_search beam +
   进图过滤。重启从 upper 快照恢复入口/上层图；无快照（未 flush）则从节点 level 扫描重建入口（退化但可搜）。
   召回@10 ≥ 0.85 维持。参数 `ef_construction`/`ef_search` 真正生效。
5. ⏳ **SIMD 距离** + 向量量化（PQ/SQ，进一步省内存）+ 邻居选择启发式 + 崩溃 durability 加固（节点记录 crc）。

## 4. 与现有件的关系

- `vecstore.rs`（现：`vectors.dat` 追加 + recover 重载）→ 演进成本设计的"向量分页存储 + 缓冲"。
- `graph.rs`（现：内存 NSW，重启 rebuild）→ 保留作小数据/测试；大数据/`open_durable` 用 `DiskGraphIndex`。
- 引擎 `GraphIndex` 接缝 + `with_graph` 注入口已就位，落盘实现直接插进去。
